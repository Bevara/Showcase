Browser-native vs. WebAssembly-pipeline JXL Load and Decode Benchmark

In this benchmark, we decode a JXL image (zoltan-tasi-CLJeQCr2F_A-unsplash.jxl) using a native browser implementation (where supported) and a Wasm-based-JXL decoding pipeline.

The native section of this demonstration will work on:

  • A chrome version (91-109) which supported JXL natively (enable JXL image format via chrome://flags/#enable-jxl)
  • The Firefox nightly build with JXL support enabled (using about:config)
  • Safari Version 17.0 or later
